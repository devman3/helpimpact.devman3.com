Overview

 

Ensure accurate, efficient and compliant tax reporting.

Use this feature to extract and export section 18A donor information as required by SARS.

 

Topics

 

Features

•Set up your export files for one or more  entities.

•Export the files per entity using the HTTPS channel.

 

Getting started

•If you have not already done so, you will need to enrol your entities for Third Party Data Submission on SARS efiling.

•If your organisation has multiple entities registered with SARS, you need to add the settings to DevMan for each one.

•The source identifier for each entity is supplied by SARS.

•Please view the tutorial below to see what additional information you will need to have on hand to complete the setup.

•Once you have added the settings, DevMan will allow you to do some test uploads via the SARS Test Platform.

•As soon as you are ready to submit your data to SARS, the support team will assist you to go live.

Processing SARS feedback files

SARS will generate a R006 batch file showing which submissions have failed.

You can import the file to DevMan on the Finance page by selecting SARS IT3 > Imports > Import file.

Once successfully imported, you can view the statistics under SARS IT3 > Imports > View import batches.

 

 

Here is an example of what the import might look like:

 

By clicking through on each section, and you will be able to open the individual records that have incorrect or missing data.

You can now decide how to proceed by fixing the errors directly or engaging with your donors to get additional information.

Once you have worked through a round of fixes, you can do a new export of all entries marked as previously failed.

SARS allows for multiple uploads.
